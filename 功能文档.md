## 1. 系统概述

**智能商品详情与营销文案生成系统** 是一套基于多模态大模型的电商内容管理平台，面向电商运营人员，利用 AI 自动生成商品详情描述、多平台营销文案和商品详情图片。

### 核心能力

| 能力 | 说明 | 使用模型 |
|------|------|---------|
| 商品描述生成 | 基于商品图片自动生成短标题、卖点、详情描述 | 通义千问 Qwen3-VL-Plus |
| 多平台营销文案 | 生成适配小红书/淘宝/抖音/通用的平台化文案 | 通义千问 Qwen3-VL-Plus |
| 商品详情图生成 | 根据商品信息生成电商详情模块图 | DashScope Wan 2.7 |
| 一键流水线 | 顺序执行描述→文案→图片三步生成 | 以上三者的组合 |
| 文案库 | 收藏优质文案为模板，跨商品复用 | — |
| 图片导出 | 单张下载 + 批量 ZIP 打包 | — |
| 多格式导出 | 文案支持 TXT/Markdown/PDF/Excel 导出 | — |

### 用户角色

| 角色 | 权限范围 |
|------|---------|
| **普通用户 (USER)** | 管理自己的商品、使用 AI 生成、文案库、导出 |
| **管理员 (ADMIN)** | 管理所有商品（只读）、审核用户、配额/权限管理、系统公告 |

## 3. 用户认证与授权

### 3.1 登录 `POST /auth/login`

- 用户名 + 密码 → BCrypt 验证 → 返回 JWT（24h 有效）
- `adminLogin: true` 时校验角色必须为 ADMIN
- PENDING/DISABLED 状态拒绝登录
- 响应含 token、userId、username、role、nickname、avatar、quotaTotal、quotaUsed

### 3.2 注册 `POST /auth/register`

- 注册后状态 PENDING（待审核），角色 USER，配额 20
- 系统上限 50 人
- 密码最低 6 位

### 3.3 修改密码 `PUT /auth/password`

- 需验证旧密码，新密码最低 6 位

### 3.4 路由守卫

- 非公开路由需要 token → 否则跳 `/login`
- `meta.admin` 路由需要 ADMIN 角色 → 否则跳 `/dashboard`
- 管理员访问 `/dashboard` 自动跳 `/admin/dashboard`

### 3.5 安全配置

- 公开路径（无需认证）：`/auth/login`、`/auth/register`、`/files/**`、Swagger 路径
- `GET /admin/announcement` 需要认证（不限角色）
- `/admin/**` 需要 ROLE_ADMIN
- CSRF 禁用，无状态 Session，CORS 允许 `http://localhost:*`

## 4. 商品管理

### 4.1 商品字段

| 字段 | 类型 | 说明 |
|------|------|------|
| name | String | 商品名称，最长 128 字符 |
| price | BigDecimal | 价格 |
| categoryId | Long | 分类 |
| stock | Integer | 库存 |
| status | String | DRAFT / ON_SHELF / OFF_SHELF |
| mainImageUrl | String | 主图（自动同步） |
| shortTitle | String | AI 精炼标题，≤15 字 |
| sellingPoints | JSON | 3-5 条卖点 |
| detailContent | Text | 150-300 字详情 |
| keywords | String | 用户补充关键词 |

### 4.2 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/products` | 分页列表，支持 keyword/categoryId/minPrice/maxPrice/status 筛选。管理员看全部，用户只看自己的。返回含 ownerUsername |
| POST | `/products` | 新建商品（DRAFT），默认名称"新商品"、价格 99、库存 100 |
| GET | `/products/{id}` | 商品详情 + 图片列表 + 解析后的卖点 |
| PUT | `/products/{id}` | 更新商品字段（仅所有者） |
| DELETE | `/products/{id}` | 级联删除商品+图片+文件（仅所有者） |
| POST | `/products/batch-delete` | 批量删除（仅所有者） |
| PATCH | `/products/batch-status` | 批量修改状态（管理员可用） |

### 4.3 权限保护

- **查看：** 管理员可查看所有商品
- **编辑/删除：** 仅商品所有者，管理员不可越权修改用户商品
- **批量上下架：** 管理员可用（管控用途）

## 6. 商品图片管理

### 6.1 上传

- 格式：JPEG/PNG/WebP，单张 ≤ 5MB，每商品最多 10 张
- 单张上传 + 批量上传
- 存储路径：`{STORAGE_ROOT}/uploads/{productId}/{uuid}_{filename}`
- 第一张自动为主图

### 6.2 排序

- 拖拽排序（vuedraggable），排序后第一张为主图
- API：`PUT /products/{id}/images/reorder`

### 6.3 删除

- 同时删除磁盘文件和数据库记录，自动重排剩余图片

### 6.4 导出

| 操作 | API |
|------|-----|
| 单张下载 | `GET /products/images/{imageId}/export` |
| 全部导出 | `POST /products/{id}/images/export-batch` → ZIP |
| 导出选中 | 同上，传入选中的 imageId 列表 |

### 6.5 访问

- `/api/files/**` 公开访问，24h 浏览器缓存，路径穿越防护

## 7. AI 商品描述生成

### 7.1 触发方式

- 商品编辑页"AI 商品描述生成"面板，选图片 + 填关键词 → 生成
- 一键流水线中勾选"描述"步骤

### 7.2 输入参数

| 参数 | 说明 |
|------|------|
| 商品图片 | 可选择特定图片，不选则用全部 |
| 关键词 | 用户手动补充 |

### 7.3 生成流程

```
图片 → Base64 → 构建 Prompt → Qwen3-VL-Plus → JSON 解析 → 写入商品 → 扣配额 → 记日志
```

JSON 解析失败 → DeepSeek 备用模型修复。

### 7.4 输出内容

| 字段 | 说明 |
|------|------|
| shortTitle | 精炼标题，≤15 字 |
| sellingPoints | 3-5 条核心卖点 |
| detailContent | 150-300 字详情 |
| category/material/color/styleTags | AI 分析的商品属性 |
| targetAudience/usageScenarios | 受众和场景 |
| complianceWarnings | 广告法合规警告 |

**API：** `POST /ai/products/{productId}/description`

## 8. AI 多平台营销文案生成

### 8.1 支持平台

| 平台 | 标识 | 风格 |
|------|------|------|
| 小红书 | `XIAOHONGSHU` | 活泼种草 + emoji + 话题标签 |
| 淘宝 | `TAOBAO` | 专业可信 + 性价比 + 卖点清晰 |
| 抖音 | `DOUYIN` | 口语化 + 短平快 + 行动指令 |
| 其它 | `OTHER` | 通用专业电商风格 |

### 8.2 触发方式

- 商品编辑页"多平台营销文案"面板 → 选平台 → 生成
- 一键流水线中勾选"文案"
- "换一换"重新生成

### 8.3 生成参数

| 参数 | 说明 | 默认 |
|------|------|------|
| platform | 目标平台 | XIAOHONGSHU |
| variantCount | 变体数量 | 1-3 |

### 8.4 前提条件

商品必须先有 `shortTitle` 和 `detailContent`（需先完成描述生成）。

### 8.5 数量限制

每个商品最多保留 3 条文案，超出的自动清理。

**API：** `POST /ai/products/{productId}/marketing-copy`

### 9.2 输入参数

### 9.4 使用指南

## 10. 一键组合生成（流水线）

### 10.1 概述

顺序执行描述→文案→图片，一次触发完成全部生成。

**API：** `POST /ai/products/{productId}/pipeline`

```json
{
  "steps": ["description", "copy", "image"],
  "platform": "XIAOHONGSHU",
  "variantCount": 3,
  "imageSectionTitle": "产品展示",
  "imageSectionCopy": "高品质材质",
  "imageVisualDirection": "简约白色背景",
  "imageAspectRatio": "3:4"
}
```

- 前一步输出自动作为后一步输入
- 每步完成后前端实时更新页面
- 某步失败不影响后续步骤

## 12. 文案导出

### 12.1 单条导出

| 格式 | API | 说明 |
|------|-----|------|

| Markdown | `GET /marketing-copies/{id}/export?format=md` | 带格式 |

| 格式 | API | 说明 |
|------|-----|------|

- 各平台独立主题色（小红书红、淘宝橙、抖音绿）

```bash
# 1. 初始化 MySQL 数据库
mysql -u root -p < backend/src/main/resources/db/schema.sql

# 2. 启动后端 (port 8080)
cd backend && mvn spring-boot:run -Dspring-boot.run.profiles=local

# 3. 启动前端 (port 5173)
cd frontend && npm install && npm run dev
```

## 24. 数据库表结构

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `sys_user` | 系统用户 | username, password_hash, nickname, phone, email, avatar, role, status, quota_total, quota_used, permissions |
| `product` | 商品 | name, price, category_id, stock, status, main_image_url, short_title, selling_points, detail_content |
| `product_image` | 商品图片 | product_id, file_path, file_name, mime_type, file_size, sort_order, is_main |
| `product_category` | 商品分类 | name, parent_id, sort_order |
| `marketing_copy` | 营销文案 | product_id, platform, variant_no, title, content, hashtags, is_favorite, is_draft, source |
| `copy_library` | 文案库 | user_id, title, platform, product_name, product_image, content, tags |
| `ai_generation_log` | AI 生成日志 | user_id, product_id, task_type, platform, model, status, duration_ms |
| `user_quota_log` | 配额变动日志 | user_id, change_by, delta, reason |
| `system_announcement` | 系统公告 | content, created_by, is_active |

所有表使用 `utf8mb4_unicode_ci`。业务表支持 MyBatis-Plus 逻辑删除（`deleted`）。
